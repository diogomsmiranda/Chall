Os seguintes testes podem ser corridos com ./chall < ./test**.in
Note que os seguintes testes verificam erros, logo irão entrar em loop uma vez que os promts irão 
ser repetidos até ser introduzido uma informação correta.

Os seguintes testam as seguintes funcionalidades / tratamento de excepções do programa:
    - test01.in: Verifica se códigos de aeroportos com letras minúsculas não são aceites.
    - test02.in: Verifica se códigos de aeroporto não têm mais de 3 letras.
    - test03.in: Verifica se a data não é colocada da forma DD/MM/YYYY.
    - test04.in: Verifica se a data não é colocada com separadores que não os aceites.
    - test05.in: Verifica se o número de passageiros é inserido de forma numérica.
    - test06.in: Verifica se é introduzido um número de sort válido.
